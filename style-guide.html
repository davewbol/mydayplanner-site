My Day Planner Style Guide

This guide defines the visual language for the My Day Planner brand across the marketing website, web app, and mobile app. Use these specifications to maintain brand unity across all platforms.

Logo usage rules: Always maintain clear space around the logo equal to the height of the owl's head. Never stretch, rotate, recolor, or place on busy backgrounds. On dark backgrounds, use the full logo as-is (it has a transparent background).

Design Principles

Friendly & Approachable

Use rounded shapes, warm colors, and the owl mascot to create a welcoming feel. Avoid sharp corners and harsh contrasts.

Clean & Uncluttered

Generous whitespace, clear hierarchy, and minimal decoration. Let content breathe. Every element should earn its place.

Cheerful Color Coding

Use the four accent colors (sky, amber, green, orange) to differentiate features, categories, and states. Never use more than two accent colors in the same view.

Consistent Across Platforms

The same palette, typography scale, and spacing system should be used on web, iOS, and Android. Platform-specific patterns (native nav bars, system fonts) are acceptable but colors must match.

Mobile App Notes

iOS (Swift / SwiftUI)

  • Use system font (SF Pro) — matches the system stack approach
  • Map Tailwind colors to SwiftUI Color assets in the asset catalog
  • Use .clipShape(RoundedRectangle(cornerRadius: 16)) for cards
  • Use Capsule() for pill buttons

Android (Jetpack Compose / Material)

  • Use system font (Roboto) — matches the system stack approach
  • Map colors to Material 3 color scheme with sky-500 as primary
  • Use RoundedCornerShape(16.dp) for cards
  • Use CircleShape or large corner radius for pill buttons

React Native / Flutter

  • Use the CSS variables above, translated to your framework's token system
  • Match border radius values: 4, 8, 12, 16, 24, 9999 (full)
  • Shadow color should match the accent color at 20% opacity
